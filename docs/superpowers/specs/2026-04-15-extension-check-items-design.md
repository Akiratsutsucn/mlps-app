# 等保 2.0 扩展题库设计文档

> 日期：2026-04-15
> 状态：已确认

## 1. 背景

现有题库 224 条，仅覆盖基础通用要求和云计算扩展。需要补充截止到 2026 年等保 2.0 的全部评测扩展项。

### 扩展标准清单

| 来源 | 扩展领域 | 标准号 |
|------|---------|--------|
| 国标原生 | 云计算（已有） | GB/T 22239-2019 |
| 国标原生 | 移动互联 | GB/T 22239-2019 |
| 国标原生 | 物联网 | GB/T 22239-2019 |
| 国标原生 | 工业控制 | GB/T 22239-2019 |
| 行业补充 | 边缘计算 | GA/T 1390.6-2025 |
| 行业补充 | 大数据 | GA/T 1390.7-2025 |
| 行业补充 | IPv6 | GA/T 1390.8-2025 |
| 行业补充 | 区块链 | GA/T 1390.9-2025 |
| 专项技术 | 5G接入 | GA/T 2348-2025 |

### 数据来源说明

- 国标原生（移动互联/物联网/工业控制）：GB/T 22239-2019 原文，数据可靠
- 行业补充 + 专项技术（2025 年标准）：基于公开资料（公安部发布说明、行业解读、相关技术规范）整理，后续拿到正式文本再修正

## 2. 数据模型变更

### CheckItem 表新增字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `extension_type` | String(50), nullable | 扩展领域，null=基础通用 | `"云计算"`, `"物联网"` |
| `standard_ref` | String(100), nullable | 标准来源编号 | `"GB/T 22239-2019"` |

### extension_type 枚举值

| 值 | 标准来源 | 分类 |
|----|---------|------|
| null | GB/T 22239-2019 | 基础通用 |
| `云计算` | GB/T 22239-2019 | 国标原生 |
| `移动互联` | GB/T 22239-2019 | 国标原生 |
| `物联网` | GB/T 22239-2019 | 国标原生 |
| `工业控制` | GB/T 22239-2019 | 国标原生 |
| `边缘计算` | GA/T 1390.6-2025 | 行业补充 |
| `大数据` | GA/T 1390.7-2025 | 行业补充 |
| `IPv6` | GA/T 1390.8-2025 | 行业补充 |
| `区块链` | GA/T 1390.9-2025 | 行业补充 |
| `5G接入` | GA/T 2348-2025 | 专项技术 |

### 迁移策略

- 启动时检测 `extension_type` 列是否存在，不存在则 ALTER TABLE 添加
- 将 `is_cloud_extension=true` 的记录更新为 `extension_type="云计算"`, `standard_ref="GB/T 22239-2019"`
- `is_cloud_extension` 字段保留但标记废弃

### EvalObjectType 新增对象类型（16 种）

| 扩展领域 | 新增对象类型 |
|---------|------------|
| 物联网 | 感知节点设备、网关节点设备 |

| 移动互联 | 无线接入设备、移动终端 |
| 边缘计算 | MEC节点、边缘网关 |
| 大数据 | 大数据平台、数据采集节点 |
| IPv6 | IPv6网络设备、IPv6安全设备 |
| 区块链 | 区块链节点、智能合约系统 |

## 4. 种子数据

### 文件结构

- 原有：`check_items.json`（224 条，不修改）
- 新增：`check_items_extensions.json`（~285 条）

### 数据结构

```json
{
  "object_type": "感知节点设备",
  "security_level": "三级",
  "category": "安全计算环境",
  "sub_category": "感知节点设备安全",
  "item_code": "L3-CES4-01",
  "content": "应保证只有授权的用户可以对感知节点设备上的软件应用进行配置或变更",
  "is_cloud_extension": false,
  "extension_type": "物联网",
  "standard_ref": "GB/T 22239-2019"
}
```

### seed_loader 加载逻辑

1. 表为空 → 导入 `check_items.json`
2. 检查 extension_type 非 null 的记录数 → 为 0 则导入 `check_items_extensions.json`

## 5. 后端 API 变更

### 查询参数扩展

`GET /api/check-items` 新增：
- `extension_type`：按扩展类型筛选，`"base"` 表示基础通用项
- `standard_ref`：按标准来源筛选

### Schema 变更

CheckItemCreate / CheckItemUpdate / CheckItemOut 新增：
- `extension_type: Optional[str] = None`
- `standard_ref: Optional[str] = None`
